# 微信红包封面调研报告（2025-2026）

## 一、制作流程与规范

### 1.1 设计规范

| 素材类型 | 尺寸 (px) | 格式 | 大小限制 |
|---------|-----------|------|----------|
| 红包封面图 | 957 × 1278 | PNG/JPG/JPEG | < 500KB |
| 封面详情页图片 | 750 × 1250 | PNG/JPG | < 300KB |
| 封面详情页视频 | 宽度≥720，宽高比 3:5~16:9 | MP4 (H.264/AVC) | < 10MB, < 15秒 |
| 封面挂件 | 按官方模板 | PNG（透明背景） | 按平台要求 |
| 气泡挂件 | 按官方模板 | PNG（透明背景） | 按平台要求 |

### 1.2 设计注意事项

- 色彩空间：RGB 模式
- 封面边缘留出至少 16px 安全区域
- 中间区域用于显示红包金额和祝福语，不可放置重要元素
- 挂件导出带透明通道的 PNG

### 1.3 红包封面组成

1. **拆红包页** — 用户看到的红包主界面
2. **红包气泡** — 聊天窗口中红包消息样式
3. **红包详情页** — 拆开后的详情展示
4. **发红包页** — 发送红包时的预览
5. **封面故事** — 详情页下拉后的品牌/创作展示页（支持图片和视频）

### 1.4 申请资格

**企业认证**：已认证的公众号

**个人认证**（满足任一）：
- 粉丝数 > 100 的公众号
- 粉丝数 > 100 的视频号
- 视频号绑定的微信表情开放平台（专辑下载量达 200 次）

### 1.5 制作工具推荐

| 类别 | 工具 |
|------|------|
| 专业设计 | Photoshop、Illustrator、Figma、Sketch |
| 在线平台 | Canva可画、稿定设计、135编辑器 |
| AI 生成 | Midjourney、DALL-E 3、Stable Diffusion、豆包/即梦、腾讯混元 |

### 1.6 费用

- 单个红包封面：1元/个
- 最低起订量：100个（100元）
- 技术服务费：20%
- 发放有效期：购买后 6 个月
- 用户使用期：领取后 3 个月

### 1.7 审核要求

- 审核周期：3-5 个工作日
- 封面名称只能使用自己主体名称中的字号或注册商标
- 设计素材需提供权属证明
- AI 生成需上传工具截图作为证明材料
- 证明材料：图片/PDF/压缩包，< 25MB

## 四、动态封面技术规格（实测验证）

### 4.1 动态封面视频规格

> 以下参数经过实际测试验证，部分与官方文档有出入。

| 参数 | 官方要求 | 实测建议 |
|------|----------|----------|
| 分辨率 | 957×1278 (3:4) | **960×1280**（H.264 要求偶数尺寸，957 是奇数会报错） |
| 编码 | H.264/AVC | H.264，profile main/high |
| 时长 | 1-3 秒 | 2.5 秒最佳 |
| 帧率 | ≤30fps | 25fps |
| 码率 | <3000kbps | **<2800kbps**（"等于 3000 都不行"，需留余量） |
| 文件大小 | <20MB | 实测 ~800KB |
| 音频 | 无要求 | 建议 `-an` 去除音轨 |
| 像素格式 | - | yuv420p（兼容性最好） |

### 4.2 H.264 编码命令（FFmpeg）

```bash
ffmpeg -y \
  -i input.mp4 \
  -t 2.5 \
  -vf "scale=960:1280:force_original_aspect_ratio=decrease,pad=960:1280:(ow-iw)/2:(oh-ih)/2" \
  -c:v libx264 \
  -preset slow \
  -crf 23 \
  -maxrate 2800k \
  -bufsize 5600k \
  -pix_fmt yuv420p \
  -an \
  -movflags +faststart \
  output.mp4
```

**关键参数说明**：
- `-crf 23`：质量控制，18 画质更好但码率容易超 3000k，23 是安全值
- `-maxrate 2800k`：严格码率上限，留 200k 余量
- `-bufsize 5600k`：码率缓冲区 = maxrate × 2
- `-an`：去掉音轨，减小文件体积
- `-movflags +faststart`：moov atom 前置，利于网络播放

### 4.3 挂件序列帧规格

| 参数 | 要求 |
|------|------|
| 格式 | PNG（透明背景） |
| 帧数 | 24-72 帧 |
| 帧率 | 24fps |
| 时长 | 1-3 秒 |

### 4.4 常见坑点

1. **957 是奇数**：H.264 编码器要求宽高为偶数，957×1278 直接编码会报 `Invalid argument`，必须用 960×1280
2. **码率不能等于 3000**：多个教程强调"等于都不行"，建议上限 2800kbps
3. **0 字节文件残留**：编码失败会留下 0 字节 MP4，下次运行会被"已存在，跳过"逻辑误判，需手动删除
4. **竖版视频**：如果源视频是横版（如即梦 AI 默认 21:9），不能直接裁剪，会丢失 70% 画面，必须重新生成竖版

## 五、AI 模型对比（实测）

### 5.1 文生图模型

| 模型 | 提供商 | 二次元效果 | 风格特点 | API 渠道 |
|------|--------|-----------|----------|----------|
| **Imagen 4** | Google (2025.5) | ★★★★★ | 赛璐璐动画、色彩鲜艳、线条清晰 | AtlasCloud / Gemini API |
| SeedDream v4 | 字节跳动 | ★★☆☆☆ | 偏 3D/CG 风格，写实感强，不适合二次元 | AtlasCloud |
| Nano Banana (Gemini 2.5 Flash) | Google | ★★★☆☆ | 对话式图片编辑，适合微调 | AtlasCloud |

**结论**：二次元风格首选 **Imagen 4**，效果远超 SeedDream v4。

### 5.2 图生视频模型

| 模型 | 效果 | 时长 | 特点 |
|------|------|------|------|
| **Kling v3.0 Pro** | ★★★★☆ | 5秒 | 动作自然，保持原画风格 |
| Seedance 1.0 | ★★★☆☆ | 5秒 | 字节跳动出品 |
| Veo 2 | ★★★★☆ | 5-8秒 | Google 出品，画质高 |

**结论**：图生视频推荐 **Kling v3.0 Pro**，原画保持度高，动作幅度适中。

### 5.3 模型参数速查

**Imagen 4 (文生图)**
```json
{
  "model": "google/imagen4",
  "prompt": "...",
  "aspect_ratio": "3:4",
  "resolution": "2k"
}
```

**SeedDream v4 (文生图)**
```json
{
  "model": "bytedance/seedream-v4",
  "prompt": "...",
  "size": "1536*2048"
}
```
> 注意：SeedDream 用 `size` 参数（如 "1536*2048"），不是 `aspect_ratio`

**Kling v3.0 Pro (图生视频)**
```json
{
  "model": "kwaivgi/kling-v3.0-pro/image-to-video",
  "prompt": "...",
  "image": "data:image/png;base64,{base64_data}",
  "duration": 5,
  "cfg_scale": 0.5
}
```

## 六、AtlasCloud API 使用指南

### 6.1 基本信息

- **API 地址**：`https://api.atlascloud.ai`
- **认证方式**：Bearer Token (`Authorization: Bearer {API_KEY}`)
- **统一端点**：`/api/v1/model/generateImage`（图片和视频生成都用这个端点）

> **重要**：不要用 `/api/v1/model/generateVideo`，会返回 400 错误。

### 6.2 请求流程（异步）

```
1. POST /api/v1/model/generateImage → 获取 prediction_id 和 poll_url
2. GET {poll_url} → 轮询等待 status == "succeeded"
3. 从 outputs[0] 下载结果
```

**响应结构**：
```json
{
  "data": {
    "id": "prediction_id",
    "status": "processing",
    "urls": {
      "get": "https://api.atlascloud.ai/api/v1/predictions/{id}"
    },
    "outputs": ["https://..."]  // 完成后才有
  }
}
```

### 6.3 可用模型列表

| 类型 | 模型 ID | 说明 |
|------|---------|------|
| 文生图 | `google/imagen4` | 最佳二次元效果 |
| 文生图 | `bytedance/seedream-v4` | 偏写实/3D |
| 图生视频 | `kwaivgi/kling-v3.0-pro/image-to-video` | 推荐 |
| 图生视频 | `bytedance/seedance-1.0/image-to-video` | 字节出品 |
| 图生视频 | `google/veo-2/image-to-video` | Google 高画质 |

### 6.4 注意事项

- 图生视频的 `image` 参数必须传 base64 Data URI 格式：`data:image/png;base64,{base64}`
- 视频生成耗时较长（2-5 分钟），轮询间隔建议 4 秒
- Kling CDN 下载可能遇到 SSL 错误，建议加 `verify=False` + 重试逻辑
- AtlasCloud 聚合了多家模型，统一接口风格，切换模型只需改 model 字段

## 七、完整制作流程

### 7.1 工作流概览

```
设计 Prompt → AI 生成底图 → 筛选/微调 → AI 图生视频 → FFmpeg 编码裁剪 → 上传微信平台
                 ↓                                    ↓
           output/anime_covers/              output/anime_dynamic_ai/
           (Imagen 4, 3:4 竖版)              (Kling v3.0 Pro, 5s→2.5s)
```

### 7.2 Step 1: 生成底图

**脚本**：`scripts/gen_anime_covers.py`

- 模型：AtlasCloud Imagen 4 (`google/imagen4`)
- 参数：`aspect_ratio: "3:4"`, `resolution: "2k"`
- 输出：`output/anime_covers_v3/` (约 4-6MB/张 PNG)
- 共 9 个 prompt，3 大风格（萌系/国风/游戏CG）

**也可用 Gemini API 直接调用**：`scripts/gen_anime_v2.py`

- 模型：`imagen-4.0-generate-001`
- 需要 `GEMINI_API_KEY` 环境变量
- 输出：`output/anime_covers/` (约 1.5-2MB/张)

### 7.3 Step 2: 生成动态视频

**脚本**：`scripts/gen_dynamic_ai.py`

- 模型：AtlasCloud Kling v3.0 Pro
- 输入：底图 PNG + 动效描述 prompt（英文）
- 输出原始视频：5 秒，约 12-15MB
- FFmpeg 裁剪编码后：2.5 秒，960×1280，<2800kbps，约 700-800KB

**动效 Prompt 示例**（水墨骑马）：
```
The girl's long black hair and crimson red sash are flowing gracefully in the wind.
The white horse is trotting forward with gentle hoof movements.
Red plum blossom petals are falling and drifting slowly from the branches above.
The golden auspicious clouds at the bottom are slowly swirling and shifting.
Subtle ink wash mist is drifting through the mountain background.
Gentle, elegant, continuous motion. Smooth camera.
```

> Prompt 技巧：描述具体的动作（什么在动、怎么动），结尾加 "Smooth camera" 避免镜头晃动。

### 7.4 Step 3: 本地假动效方案（备选）

**脚本**：`scripts/gen_dynamic_covers.py`

用 Pillow 逐帧渲染 + FFmpeg 编码，实现简单的 Ken Burns（缩放+平移）+ 粒子效果：
- 花瓣飘落（Petal）
- 雪花飘落（Snowflake）
- 金色闪光（Sparkle）
- 暖色光斑（Bokeh）

> 效果远不如 AI 图生视频，画面内容不会真正动起来，只有缩放平移+粒子叠加。

### 7.5 已生成的成品

**底图（v1, Gemini API, 18张）**：`output/anime_covers/`
- 01 萌系少女 × 3、02 萌马 × 3、03 国风水墨 × 3
- 04 新中式 × 3、05 游戏CG × 3、06-08 其他

| 入口 | 地址 | 适用对象 |

1. 开通视频号并积累 100+ 粉丝
2. 进入视频号创作者中心

1. **视频号直播/视频**：在视频号发布动态封面展示视频，粉丝通过互动领取

3. **委托有资质的账号**：找已认证的公众号代为上传

## 九、设计方案与 Prompt 库

### 9.1 三大风格方向

| 风格 | 代表作品 | 特点 | 目标用户 |
|------|---------|------|---------|
| 萌系 | Q版少女拜年、萌马贺岁 | 大眼萌系、暖色调、可爱治愈 | 女性用户、泛二次元 |
| 国风融合 | 水墨仙侠骑马、少女抚琴 | 水墨+动漫、留白构图、红金点缀 | 国风爱好者、文化向 |
| 游戏CG | 金发战姬、仙女御马 | 高细节、戏剧光影、华丽装饰 | 核心二次元、游戏玩家 |

### 9.2 Prompt 模板

**通用结构**：
```
[角色描述] + [服装/装饰] + [动作/姿态] + [背景场景] + [氛围元素(烟花/花瓣/雪花)] + [风格标签]
```

**风格标签参考**：
- 萌系：`cel-shaded kawaii illustration, vibrant saturated colors, highly detailed anime art`
- 国风：`Chinese ink wash painting style, ink splatter textures, elegant negative space, red and gold accents`
- 游戏CG：`game promotional CG art style, dramatic cinematic lighting, rich saturated colors, highly detailed`

### 9.3 动效 Prompt 模板

```
[具体物体] is [动作ing] [方向/方式].
[另一物体] is [动作ing] [细节].
...
[整体氛围]. Smooth camera.
```

关键点：
- 用**现在进行时**描述每个动作
- 每个句子只描述一个动态元素
- 动作要具体（"flowing in the wind" 比 "moving" 好）
- 结尾加 "Smooth camera" 或 "Static camera" 控制镜头
